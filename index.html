01 — Documents · 환자 후기 AI 분류·분석 + 문서 작성 역량

말보다 결과물로 확인하실 수 있게 정리했습니다

세마컨설팅 CS 연구원 공고의 "자료 리서치 + 보고서 작성 + CS 모듈 정성·정량 자료 분석" 항목에 대해 두 가지 자료를 첨부합니다. ① 본 지원을 위해 새로 작성한 가상 ABC 정형외과 환자 후기 100건 AI 분류·분석 보고서 12장. ② 학부 시절 작성한 정리자료 중 문서화 방식이 잘 드러나는 페이지를 압축한 8쪽 샘플.

Primary · Created for SEMA Consulting Application

가상 ABC 정형외과 환자 후기 100건
AI 분류·분석 + 인사이트 보고서

가상 ABC 정형외과의원(개원 5년차, 일평균 외래 80명) 대상 환자 후기 100건 AI 분류·분석. 세마컨설팅의 PEI 솔루션과 연결 가능한 환자후기 분류·분석 흐름을 외부인의 입장에서 가상 데이터로 구성해본 결과물입니다. 네이버 플레이스 40건 + 카카오맵 30건 + 자체 홈페이지 30건의 출처 분포를 가정했고, Claude Opus 4.7으로 7개 카테고리 × 3단계 감성 분류 후 무작위 표본 20건을 수동 검증해 일치율 88%를 확인했습니다. 모든 후기와 수치는 가상 시나리오 기반이며, 실제 ABC 정형외과 또는 세마컨설팅 고객사와 무관함을 본문에 명시했습니다.

Analysis Flow · 분석 흐름 다이어그램

12장 보고서의 분석 흐름을 한 장으로 압축한 다이어그램. AI 1차 분류 + 사람 검증 + 부정 후기 안전 게이트의 3단 구조.

  1. Input 환자 후기 100건 네이버 플레이스 40 · 카카오맵 30 · 자체 홈페이지 30
  2. Stage 1 · 카테고리 정의 7개 카테고리 × 3단계 감성 사전 정의 의료진 / 직원 / 진료 / 대기 / 시설 / 비용 / 예약
  3. Stage 2 · AI 1차 분류 Claude Opus 4.7으로 100건 분류 + 감성 부여
  4. Stage 3 · Human 검증 무작위 20건 표본 수동 검증 (일치율 88% 확인) + 부정 후기 100% 재확인 (안전 게이트)
  5. Stage 4 · 3축 분석 카테고리별 부정 비율 + 키워드 빈도 + 후기 간 동시 언급 패턴
  6. Output · CS 개선 제안 우선순위 3가지 + 검증 지표(KPI)

02 — Decisions · 12장 보고서를 만들며 내린 핵심 판단

같은 분석을 만드는 방법은 많지만,
어떤 판단은 작업하는 동안에만 보입니다

첨부한 12장 보고서를 작성하며 내린 핵심 설계 판단 세 가지를 정리합니다. "더 화려하게 보이는 것"이 아니라 "신입 연구원의 정직성 신호를 남기는 것"이 매번의 기준이었습니다.

01

왜 액션 제안을 "예상 효과 %"가 아닌 "검증 지표(KPI)"로 정리했나?

Context

가상 데이터 기반 보고서에서 "재방문 부정 후기 50% 감소" 같은 단정적 효과 수치는, 실제 운영 효과를 검증할 수 없는 상태에서 신뢰를 떨어뜨릴 위험이 있습니다.

Decision

모든 액션 제안을 "검증 지표"로 정리. 재방문 의향 부정 후기 비율 변화, 부정 키워드 빈도 변화, 부정 후기 탐지 후 내부 공유까지의 소요시간 등 실제 데이터에서 추적 가능한 KPI 형태로 제시.

Trade-off

임팩트 있는 효과 수치 부재 ↔ 신입 연구원으로서의 정직성 + 컨설팅 보고서의 추적 가능한 톤.

02

왜 실제 병원이 아닌 가상 ABC 정형외과 시나리오를 만들었나?

Context

실제 병원 환자 후기 데이터는 외부인이 접근할 수 없고, 임의의 실병원 이름을 쓰면 해당 병원에 의도하지 않은 영향을 줄 수 있습니다. 그러나 분석 흐름은 구체적인 시나리오 위에서만 시연 가능합니다.

Decision

정형외과 일반 후기 패턴을 참고한 가상 ABC 정형외과의원 시나리오(개원 5년차, 일평균 외래 80명)를 설정. 모든 슬라이드 풋터에 "Sample · 가상 시나리오" 라벨을 박고, 4가지 한계(가상 데이터 / 표본 편향 / 단일 시점 / AI 오차)를 별도 슬라이드로 분리해 명시.

Trade-off

실데이터 기반 신뢰도 부재 ↔ 분석 방법론 시연 + 윤리적 안전성.

03

왜 자소서 한 편이 아니라 4종(자소서 + 12장 + 8쪽 + 페이지) 다층 구조로 첨부했나?

Context

병원 도메인 직접 경험이 없는 지원자가 글만으로 자기 자랑하는 것보다, 실제 산출물로 직무 적합성을 검증받는 것이 채용 담당자에게 더 명확한 신호를 줍니다.

Decision

4종 다층 구조 — ① 본 페이지(5분 훑기), ② 12장 보고서(직무 시연), ③ 8쪽 HWP 샘플(장기 문서화 증거), ④ 자소서(경력 서사). 채용 담당자가 시간이 없으면 ①만, 더 보고 싶으면 ②③④로 깊이 들어가는 동선.

Trade-off

작성 시간 부담 + 분량 과잉 위험 ↔ 자료 자체가 직무 시연 산출물 + 채용 담당자 동선 존중.

06 — Domain · 정성·정량 자료 분석 + 외부 기관 보고 도메인

"데이터 정리 → 인사이트 → 의사결정자 보고"를
7년간 도메인을 바꿔가며 반복했습니다

세마컨설팅 CS 연구원의 핵심 업무 — "자료 리서치 + 보고서 작성 + CS 모듈 정성·정량 분석 + 회의 동행"의 각 영역에 본인의 7년 경력이 어떻게 닿아 있는지 정리했습니다. 병원 도메인 자체는 입사 후 학습이 필요한 영역입니다.

공고에 명시된 4가지 핵심 업무 각각에 대응하는 본인의 경력 매핑. EXP #2 ★(파란 박스)는 첨부한 12장 보고서로 시연한 산출물입니다.

  1. JD #1 — 자료 리서치 + 보고서 작성 ↔ 한화오션 3년 4개월 외부 기관 보고 (월 단위 반복)
  2. JD #2 — CS 모듈 정성·정량 자료 분석 ↔ 12장 환자후기 보고서 (100건 × 7카테고리 × 3감성, 일치율 88%)
  3. JD #3 — 컨설턴트 보조 + 회의 동행 ↔ 대치동 4년 3그룹 의사결정자 보고 (학생/학부모/원장)
  4. JD #4 — 교육·실습 자료 작성 ↔ 한서대 학습자료 + 첨부 8쪽 샘플
2022.04 — 현재 · 대치동 정시 컨설팅 · 프리랜서 컨설턴트

학생·학부모 데이터 관리 + 대면 컨설팅 · 매 시즌 수백 명

스프레드시트 기반 정성·정량 데이터 운용. 매 시즌 100명 이상의 학생 모두에 대해 정량 데이터(점수·등급·지원 학교·합격 가능성)와 정성 데이터(성향·가족 의사결정 구조·진로 의지)를 직접 시트로 관리·업데이트했습니다. 같은 데이터를 학생·학부모·원장 세 그룹에 다른 무게로 정리해 전달하는 작업이 일상이었고, 이게 병원 내 다양한 의사결정자(원장·실장·직원)에게 환자 응답 데이터를 각기 다른 무게로 옮기는 작업과 본질적으로 같다고 봅니다.

구조화된 4단계 컨설팅 사이클. 사전 설문 → 대면 컨설팅 → 후속 관리 → 파이널 콜로 이어지는 4단계 프로세스를 매 시즌 반복 적용했습니다. 사전 설문은 정성 데이터 수집, 대면 컨설팅은 정량·정성 종합 인사이트 전달, 후속 관리는 데이터 업데이트와 추가 분석으로 구성됩니다. 세마 CS 모듈의 컨설팅 사이클(진단 → 매뉴얼 → 교육·코칭 → 개선)과 단계 구조가 닮아 있다고 봅니다.

검증된 설득력. 학원 재등록과 지인 추천이라는 결과로 컨설팅 품질이 검증되어 왔습니다. 학부모·학생·원장 = 고관여 의사결정자 응대 경험은, 병원 원장·실장·직원 등 다층 의사결정자에게 인사이트를 전달하는 자리에서도 같은 근육이 작동한다고 봅니다.

2014.03 — 2017.06 · 한화오션(주) 특수선무장설계팀 · 3년 4개월

장보고-Ⅲ 잠수함 프로젝트 · 외부 기관 보고 자료 설계

국방과학연구소 보고 자료 작성. 부서 내 여러 업무별 담당자들로부터 자료를 취합해 외부 기관 보고용 자료로 재구성하는 작업을 매월 반복했습니다. 의사결정자 한 명이 짧은 시간 안에 핵심을 파악할 수 있는 형태로 자료의 순서·무게·시각화를 재설계하는 일이 이 시기의 주된 업무였고, CS 연구원의 "컨설턴트 요청 업무 보조 + 회의 동행 + 회의록 정리" 흐름과 결이 같습니다.

방산 보안 환경에서 익힌 데이터 신뢰성 감각. 사내 데이터 보안·외부 반출 정책·기관 간 데이터 공유 제약 등을 실무에서 익혔습니다. 이 감각은 첨부 12장 보고서에서 "가상 데이터 / 표본 편향 / 단일 시점 / AI 오차" 4가지 한계를 별도 슬라이드로 분리해 명시한 부분에 그대로 옮겨졌습니다. 환자 데이터를 다루는 컨설팅 환경에도 같은 결로 적용 가능합니다.
